Wij creëren Front-End Developers

Minor Web Development is voor studenten die zich willen focussen op de beroepsrol Front-End Developer.

Inhoud van de minor

Intro

De minor web development wordt gegeven door Communication and Multimedia Design aan de HvA. In deze minor leer je interactieve toepassingen te maken met behulp van de webtechnologiën: HTML, CSS en JavaScript.

In verschillende vakken en projecten leer je over browser technologieën, het ‘real time web’, web design, performance, rapid prototyping, interaction engineering, documenteren, versiebeheer, debuggen, testen, sockets, responsive, reactive, micro libraries, API’s en meer ...

“De minor is echt geweldig. Je leert in een half jaar enorm veel over verschillende technologieën waarmee je responsive real-time websites kan maken. Ik zou de minor zo weer kiezen!”

Mohammed Mulazada

Inhoud van de minor

Een semester lang gaan we de grenzen van het web opzoeken. Dagelijks zitten we van half 10 tot 5 in onze eigen ‘studio’. 3 weken lang krijg je 2 vakken en een overkoepelend project van een week waarin je de skills die je geleerd hebt tijdens de vakken gaat uitoefenen. Na elk project starten er weer twee nieuwe vakken, met ook weer een project. Dit gebeurt in totaal 3 keer.

Nadat alle zes vakken en drie projecten geweest zijn begint de meesterproef: een project van 5 weken waarin je al je geleerde skills in een groot project gaat gebruiken.

    De 6 vakken zijn:
  • Web Apps From Scratch
  • CSS To The Rescue
  • Browser Technologies
  • Performance Matters
  • Web Design
  • Real-Time Web
“Ik vond het erg fijn dat je elke 3 weken een vak krijgt, met daarna een project waarbij je de stof behandeld die je in de vakken hebt gekregen. Zo leer je hoe alles wat je geleerd hebt ook daadwerkelijk in het echt kan gebruiken.”

Rick Buter

Vakken minor

Web Apps From Scratch

Intentie van het vak

In dit vak gaan we een web app maken zonder frameworks of onnodige libraries, dus zoveel mogelijk met native HTML, CSS & JavaScript. Het eindproduct is een modulair opgezet prototype voor een single page web app. Data wordt opgehaald uit een externe API, waar nodig gemanipuleerd en vervolgens getoond in de Web App. Je leert hoe je structuur aanbrengt in je code en hoe je hiermee 'from scratch' een (prototype voor een) web app maakt. Met deze kennis begrijp je daarnaast beter hoe bestaande api’s, libraries en frameworks werken.

Docenten

Joost Faber & Laurens Aarnoudse

Leerdoelen

  1. Student kan JavaScript OOP en/of Functional Programming principes toepassen (minimaal object literal pattern)
  2. Student kan met JavaScript data ophalen vanuit een API, manipuleren en tonen in een web app
  3. Student kan met JavaScript interactie toevoegen aan een web app
CSS To The Rescue

Intentie van het vak

In dit vak gaan we aan de slag met CSS. We gaan goed werkende, plezierige responsive oplossingen bedenken én maken voor complexe interfaces. Dat is erg belangrijk, te veel ontwerpers kunnen dit niet zo goed. Het is ook belangrijk om een aantal basisprincipes achter CSS goed te begrijpen. Niet alleen op praktisch niveau, ook op experimenteel niveau. Zonder goed begrip van de basisprincipes is CSS magisch en weird. Met een goed begrip heb je CSS onder controle en kan je het laten doen wat jij wil. En dat is nodig om webpagina’s vorm te geven met attention to detail. Webpagina’s waar mensen blij van worden.

Docenten

Vasilis van Gemert & Titus Wormer

Browser Technologies

Intentie van het vak

Een van de mooiste principes van het web is dat het er echt is voor iedereen. Iedereen met een computer en een browser moet gebruik kunnen maken van het web. Het web is geen gecontroleerde (programmeer) omgeving. Je kan er gerust van uit gaan dat niemand precies hetzelfde te zien krijgt als wat jij ziet in jouw browser. Er zijn natuurlijk de technische beperkingen. Zoals - Afmetingen van de browser - Grootte van het apparaat - Manier van interactie - Kwaliteit van de hardware - Kwaliteit van het netwerk. En er zijn mensen. Allemaal verschillende mensen ...

In het vak Browser Technologies leer je hoe je goede, robuuste, toegankelijke websites maakt. Je gaat leren over Progressive Enhancement, Feature Detection en Fallback. Het web is er voor iedereen. In dit vak leer je hoe je daarvoor kan zorgen.

Docenten

Koop Reijnders & Krijn Hoetmer

Performance Matters

Intentie van het vak

In deze drie weken gaan we van bestaande web applicaties de performance verbeteren. We werken aan HTML, CSS en JavaScript optimalistaties en hoe we het HTTP protocol beter kunnen benutten.

Docenten

Joost Faber & Titus Wormer

Leerdoelen

  1. De student begrijpt de termen critical rendering path en perceived performance
  2. De student kan verschillende tools inzetten om de performance van web apps te meten, en om optimalisaties te documenteren
  3. De student kan op code niveau optimalisaties doen tbv een betere performance van web apps
  4. De student begrijpt het verschil tussen server side en client side rendering en wat het effect is op de performance van web apps, als deze worden gecombineerd
  5. De student kan een task manager inzetten tbv een betere performance van web apps
  6. De student kan een service worker inzetten tbv een betere performance van web apps
Web Design

Intentie van het vak

In dit vak gaan we rapid prototypen en testen met echte gebruikers. Is er goede interactie? Zijn er goede design patterns gekozen? Zijn de basis regels van user interface design goed toegepast? Is er een focus op de core functionaliteit?

Docenten

Koop Reynders, Joost Faber & Vasilis van Gemert

Leerdoelen

  1. Interface design principles snappen en toepassen
  2. Leren hoe je moet (Usability) testen en de resultaten gebruiken voor het verbeteren van je ontwerp
  3. User needs begrijpen en gebruiken in je ontwerp
Real-Time Web

Intentie van het vak

De komende drie weken leer je hoe je real time/ live data op een inzichtelijke manier toegankelijk kunt maken. Terwijl je bij het vak ‘Web of Things’ leert hoe je sensor data uit “slimme” devices kunt lezen en deze devices zelfs kunt aansturen, ga je bij RTW precies dit soort data real-time inzichtelijk en toegankelijk maken voor eindgebruikers.

Docenten

Laurens Aarnoudse & Titus Wormer

Leerdoelen

  1. Student kan zelfstandig een (Node) webapp bouwen met een backend die gebruikt maakt van templating, routing en een remote database.
  2. Student kan een live applicatie opleveren waar interactie mee mogelijk is die invloed heeft op de inhoud van de applicatie via manipulatie van de database.
  3. Student kan live data vertalen naar een reactive view (datavisualisatie mag) op een manier die voor een eindgebruiker zinnig is.
